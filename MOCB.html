Electronic Codebook — ECB
Deterministic

Each plaintext block is encrypted independently using the same key. No chaining, no IV. The simplest block cipher mode — and the most dangerous for structured data, because identical plaintext blocks always produce identical ciphertext blocks, leaking patterns.

Cipher Block Chaining — CBC
Probabilistic

Each plaintext block is XORed with the previous ciphertext block before encryption. An Initialisation Vector (IV) starts the chain for the first block. Identical plaintexts produce different ciphertexts as long as the IV changes — fixing ECB's biggest flaw.

Cipher Feedback — CFB
Stream-like

CFB turns a block cipher into a self-synchronising stream cipher. The previous ciphertext block is encrypted, and the result is XORed with the current plaintext. Encryption is sequential; decryption can be parallelised. Any single-bit error in a ciphertext block affects only that block and part of the next.

Output Feedback — OFB
Stream-like

OFB generates a keystream independent of plaintext or ciphertext — the encryption function is applied repeatedly to the IV to produce keystream blocks, which are then XORed with plaintext. Errors don't propagate. The keystream can be pre-computed. Re-using an IV is catastrophic.

Counter Mode — CTR
Parallelisable

CTR combines a nonce with an incrementing counter to produce unique inputs for each block. Each block is encrypted independently — making CTR fully parallelisable in both directions. It turns the block cipher into a synchronous stream cipher. Random access decryption is possible.

Galois/Counter Mode — GCM
AEAD

GCM = CTR mode encryption + GHASH authentication. It produces an authentication tag alongside ciphertext, guaranteeing both confidentiality and integrity. Additional Authenticated Data (AAD) can be authenticated without being encrypted. The industry standard for TLS 1.3, SSH, and modern protocols.

// GCM — conceptual note
GCM uses Galois field (GF 2¹²⁸) multiplication for GHASH — not simulatable with simple ASCII operations. In real use: AES-256-GCM with a random 96-bit nonce is the recommended choice. Key properties to remember: • Nonce must NEVER be reused with the same key — catastrophic if so. • Auth tag (128-bit) detects any tampering with ciphertext or AAD. • Both encrypt and decrypt can run fully in parallel. • Used in: TLS 1.3, SSH, IPSec, WireGuard, Google QUIC.
